Requirements

Priority Order of Requirements

1. Work with other teams on the desktop and mobile applications
Communicate and develop with the other Delta V Innovations teams that are working on the desktop and mobile applications with the goal in mind for future implementation. Our goal will be to have a complete project encompassing these requirements for presentation in Marksbury Theatre by the end of the Spring 2018 semester.

2. Have a database within the cloud
The primary goal of the project is to have a database stored within the cloud. This will allow for database communication with all systems neccessary for essential crash scene investigation.
Use Microsoft Azure or Amazon AWS
Effort and Size Estimation: Creating a database within the cloud should be a fairly simple task. Microsoft offers their Azure cloud storage and Amazon offers its AWS. Creating a database within the cloud should be as simple as signing up for one of these services.

3. Internet Communication protocols to the desktop software and mobile device for data sent to clients, and storage of data recieved from clients
Communication protocals must be established to ensure that data can be sent from mobile devices in the field and properly stored in the cloud database. Additionally, communication protocals must also be developed for the desktop aplication side allowing it to properly communicate with the cloud application and pull essential crash scene information needed for scene development.
Effort and Size Estimation: To develop communication protocols our team must research the system we decide to use (Microsoft Azure vs Amazon AWS) and the way data can be stored and accessed within their cloud. Doing this research and correctly designing protocols to follow their established standards could take a couple days although there should be plenty of information avaliable as to how this is done.
Microsoft Azure Database Documentation
Amazon AWS Database Documentation

4. Database created with typical tables and/or fields needed for data for the initial functions to be developed
The cloud database must be created and developed with initial relational tables and their essential corresponding attributes being designed. Note that as the project progresses more and more tables and the overall schema behind the database will most likely change.
Effort and Size Estimation: Designing the initial database and tables will be a week or two process as it will involve back and forth communication with the customer to ensure our design meets the information requirements they have as the database we develop will be a large system. Making additons to the database per customer requests throughout the semester will also be necessary.
Microsoft Azure Database Database Creation
Amazon AWS Database Creation

5. SQL table updates to insert initial data
The team will have access to crash scene data from the past 20 years from the Cincinnati area as guaranteed from the customer. SQL queries must be developed to ensure this data can accurately and securely be transferred from these servers to the cloud database. Additionally, the customer has a series of tables with friction gradients, car information, etc. that must be uploaded to the cloud database. SQL queries to transfer this information from the current format to the cloud database must be designed.
Effort and Size Estimation: This part of the project may very well be the most difficult and could take a couple weeks or even a month to accompish. The customer currently has 3TB of data stored on a police server that must be trasnferred over as well as tables of information in several different formats that must be uploaded. Correctly storing and transferring this information to the cloud may be challenging.

6. SQL queries for data for the first functions to be developed
SQL queries must be designed that allow for the first functions of the mobile and desktop applications to be tested. This queries will function by using the communication protocols established in step 2. to send data from the mobile app to the cloud server and consequently pull information from the cloud server to the desktop application. Initial SQL queries must be designed to ensure information is stored and accessed accurately, quickly, and effeciently.
Effort and Size Estimation: Developing initial queries will take a week or two depending on the involvement of the query. Note this will also be an ongoing process as more functionalities are added to the mobile and desktop app as the semester progresses.
SQL Queries

7. Security protocols to restrict access to data and programs (starting with user ID and password)
Security protocols must be designed to ensure that the proper users/individuals have access to only the neccessary data they need to perform their duties. Protocols must also be develped to ensure updates or changes to the database cannot be made without the proper access clearences.
Effort and Size Estimation: This step will most likely be a week long process and again will be something that will constantly be evaluated throughout the semester to meet customer needs. Our group will seek to implement aspects of Discretionary Access Control

8. Other initial functions viewed as important by the customer
a. Create a method that permits an easy update of database (IE Vehicle Database) annually or as updates become available to the company.
b. As database updates are installed, allow for pushing of data back to the mobile and desktop applications.

Effort and Size Estimation

This project will require every group member to work diligently for several hours per week. Each member will be required to work on thier own as well as within their group to complete each of the goals that we have established below. For the purpose of this project our end goal is to complete a proof of concept to our customer to show the functionality of our database with respect to the mobile and desktop applications. This project will take a minimum of ten weeks to implement. The cohesiveness between the three different project groups will ultimately decide when this project will be complete.
